
# Contributing to rSOFA

We welcome contributions to the rSOFA package, a robust R package designed for calculating Sequential Organ Failure Assessment (SOFA) scores from datasets. This tool supports both cross-sectional and longitudinal analyses, accommodating datasets with repeated measurements.

- **Create a new branch**: for each new feature or bug fix, create a new branch from `main`.

- **Document your changes**: update the README.md or any relevant documentation with details of your changes.
- **Submit a pull request**: once your feature is ready and tests are passing, submit a pull request (PR) to the main rSOFA repository.

## Coding standards ✨

- **Style guide**: follow the [tidyverse style guide](https://style.tidyverse.org) for R code to maintain consistency across the package.
- **Documentation**: use [Roxygen2](https://cran.r-project.org/package=roxygen2) for documenting functions. Ensure that all functions are well-documented, with clear descriptions, parameters, and return values.

## Reporting bugs 🐛

If you encounter any bugs or issues, please open an issue on GitHub with a clear description of the problem, steps to reproduce it, and any error messages. Include screenshots if applicable.

## Feature requests 💡

Have ideas for new features or enhancements? Open an issue on GitHub detailing your suggestion, its potential benefits, and any preliminary thoughts on implementation.

## Questions or discussions 💬

For questions or discussions related to development or usage of the rSOFA package, please use the GitHub Discussions feature. This platform allows for open dialogue and community support.

## Acknowledgements 🎉

Contributors like you help make rSOFA a valuable tool for the medical research community. We appreciate your efforts and look forward to your contributions.
